# Server Docs

Quick reference for the Express backend.

---

## Stack

| Piece | Package | Notes |
|---|---|---|
| Server | Express 4 | Runs on `PORT` env var, falls back to `3001` |
| DB client | `postgres` (porsager) | Connects via `DATABASE_URL` env var, falls back to local Postgres.app |
| Database | PostgreSQL 18 (local) | Supabase (production) — same engine, swap connection string only |

---

## File Map

| File | Responsibility |
|---|---|
| `index.js` | Express app entry point. Mounts all routes, starts the server. |
| `db.js` | Creates and exports the `postgres` client instance (`sql`). Import this wherever you need a DB query. |
| `uid.js` | Generates human-readable user IDs (e.g. `Vivian_Hurley_1714400000000_042`). Used when creating a new session. |
| `schema.sql` | One-time setup script. Run with `psql macro_database -f server/schema.sql`. |
| `server_docs.md` | This file. |

---

## Database Schema

| Table | Purpose |
|---|---|
| `sessions` | One row per app session. Holds a generated `user_id` and start timestamp. |
| `domains` | Known domains used to classify links (matched via `domain_identifier` substring). |
| `macros` | URL parameter find-and-replace rules (`macro_title` = param name, `macro_pre` = token to find, `macro_post` = replacement). |
| `links` | Every unique URL extracted from uploaded VTS files. `full_link` has a UNIQUE constraint — duplicate URLs are rejected at the DB level. |
| `link_macros` | Junction table linking `links` ↔ `macros` (many-to-many). |

---

## Key Conventions

- **All IDs are UUIDs** generated by PostgreSQL (`gen_random_uuid()`). `sessions.user_id` is the exception — it's a human-readable string from `uid.js`.
- **Duplicate links** — inserting a `full_link` that already exists returns the existing row rather than throwing. The API response includes a `duplicates` array so the caller knows what was skipped.
- **No CORS config needed** — Vite proxies `/api/*` to `localhost:3001` in dev. In production, Express and the built frontend will be served from the same origin.
- **Environment variables** — `PORT` and `DATABASE_URL`. Neither is required locally; both should be set in the production environment.
